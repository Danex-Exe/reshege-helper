ReshEge-Helper: как добавлять свои кнопки и менять поведение

1) Где добавлять кнопки в выдвижное меню
В файле script.js есть точка расширения:

const EXTRA_MENU_BUTTONS = [];

Добавляй туда объекты кнопок. Каждая кнопка поддерживает поля:
- id: уникальный строковый идентификатор
- label: подпись кнопки
- icon: иконка (обычно emoji или символ)
- onClick: функция, которая вызывается при клике

Пример:

const EXTRA_MENU_BUTTONS = [
  {
    id: 're_refresh_data',
    label: 'Обновить данные',
    icon: '🔄',
    onClick: ({ menu }) => {
      menu.historyData = null;
      menu.highscoreData = null;
      alert('Кэш очищен. Открой таблицу или историю снова.');
    }
  },
  {
    id: 're_open_help',
    label: 'Открыть справку',
    icon: '❓',
    onClick: () => {
      window.open('https://ege.sdamgia.ru/', '_blank', 'noopener');
    }
  }
];

Важно:
- id не должен повторяться (иначе кнопка не добавится).
- onClick должен быть функцией.
- Для доступа к состоянию меню используй объект menu из onClick.

2) Как добавлять кнопки без перезаписи массива
Скрипт публикует API в window.ReshEgeHelper:

window.ReshEgeHelper.addMenuButton({
  id: 're_custom_runtime',
  label: 'Runtime кнопка',
  icon: '🧪',
  onClick: ({ menu, game }) => {
    console.log('Текущий view:', menu.currentView);
    console.log('Объект игры:', game);
  }
});

Это удобно, если ты подключаешь отдельный пользовательский скрипт поверх основного.

3) Что уже есть в контексте onClick
В callback передается объект:
- menu: экземпляр MenuController
- game: текущий объект игры

Часто полезные поля:
- menu.historyData
- menu.highscoreData
- menu.currentView

Часто полезные методы:
- menu.renderMainMenu()
- menu.showHistory()
- menu.showLeaderboard()
- menu.openDrawer()
- menu.closeDrawer()

4) Как менять стили сайта
Основные стили страницы задаются в функции applySiteStyles() в script.js.
Там удобно менять:
- отступы игрового окна (.game_div, .game_prob, .game_answer)
- стиль кнопок (.game_turn_bet, .game_answer_send, .game_find_player, .game_hist_back, .game_high_back)
- цвета текста, фоны, тени и адаптив

Рекомендация:
- сначала меняй CSS-переменные в блоке :root (цвета/ширина drawer),
- потом точечно корректируй конкретные селекторы.

5) Как менять стили и поведение самого drawer-меню
Стили меню находятся в MenuController.createStyles().
Там меняются:
- кнопка-ярлык открытия меню (.ReshEge_Helper_menu_toggle)
- панель drawer (.ReshEge_Helper_menu_drawer)
- кнопки в меню (.ReshEge_Helper_menu_button)
- кнопка Назад (.ReshEge_Helper_back_button)
- таблица лидеров (.leaderboard-table)

Логика меню:
- регистрация кнопок: registerMenuButton()
- главное меню: renderMainMenu()
- история: showHistory() / renderHistory()
- лидеры: showLeaderboard() / renderHighscore()

6) Безопасное редактирование (чтобы не ломать данные)
- Если вставляешь пользовательские строки в HTML, используй escapeHtml(...).
- Не удаляй patchGameMessageHandler(): через него меню получает show_history/show_highscore.
- Если меняешь разметку кнопки Назад, проверь, что id="re_back_btn" либо обнови обработчик.
- После изменений обязательно обнови страницу и проверь:
  1. открытие/закрытие drawer,
  2. отображение таблицы лидеров,
  3. отображение истории,
  4. работу кнопки Назад,
  5. мобильный вид.

7) Где смотреть ошибки
- Открой DevTools (F12) -> Console.
- Сообщения скрипта начинаются с [ReshEge-Helper].
- Если пользовательская кнопка не появилась, сначала проверь уникальность id и тип onClick.
